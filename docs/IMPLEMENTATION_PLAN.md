# 구현 계획

1. 첨부파일 `policy-rag.zip`, `policy_collection.md`, `policy_search.md`, `final_요정서_0709.xlsx`, `final_ERD.png`를 확인한다.
2. 기존 저장소 루트에서 `.git`을 건드리지 않고 Spring Boot 3.5.16 / Java 17 / Gradle 프로젝트를 생성한다.
3. ERD 기준 Flyway 스키마를 먼저 만들고, 원본 수집 및 임베딩 동기화를 위한 운영 테이블을 별도로 추가한다.
4. 네 개 출처별 정책 수집기, 원본 응답 저장, 정규화, 중복 처리, 수집 로그를 구현한다.
5. 규칙 기반 조건 추출, 선택적 RAG 검색, MySQL 대체 검색, 엄격한 지역 필터, 신청 가능성 판별, 관심 정책, 캘린더, 상세 API를 구현한다.
6. Thymeleaf 화면으로 검색, 정책 상세, 관심 정책, 캘린더 화면을 구현한다.
7. 실제 외부 API 키 없이 실행 가능한 단위 테스트와 통합 테스트 기반을 추가한다.

현재 상태: 저장소에 구현 완료. 실제 외부 API 키와 실 API 응답 검증은 로컬 비밀 설정이 필요하다.
